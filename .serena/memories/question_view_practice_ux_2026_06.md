# Question view and Practice Mode UX decisions from 2026-06 work

- In the desktop question view, metadata should be compact and side-by-side where possible: Topic, Concept, Cognitive Stage. On mobile it can stack.
- The answer-choice area should have a subtle divider above it. The prompt (`Your choice`/localized label) and horizontal radio buttons should be centered relative to the question frame, not the viewport edge. Radio spacing was increased for scanability.
- On question changes, the question view should scroll back to the top. Track the actually rendered `frage_idx`, not only `_current_question_idx`, because some navigation paths set `_current_question_idx` before render.
- In Practice Mode, after answer submission and immediate feedback, scroll to the Next button area rather than to the top. A dynamic bottom spacer is needed after visible feedback so short explanations still leave enough scrollable page height for the Next button to reach the intended position.
- Streamlit top chrome/header can cover elements if the target offset is too small. Use about `96px` top offset for Next-button and question-expander scroll targets; `24px` was too high and caused clipped titles/buttons.
- Opening question expanders should pull the expander heading upward with a DOM-side toggle listener scoped to the question view. Limit this behavior to the question view so start page, sidebar, and summary expanders are not unexpectedly moved.
- The detailed-explanation button is a special case: it triggers a Streamlit rerun and opens an expander already expanded, so set a one-shot session flag and scroll to an anchor before the expander after rerender.
